## Railway Configuration

### Step 1: Authenticate Railway

```bash
railway login
# Opens browser → authenticate with GitHub
```

### Step 2: Initialize Railway Project

```bash
cd C:\Proyectos\00Velkor
railway init --name velkor-platform
```

### Step 3: Create Services in Railway

**Option A: Using Railway Dashboard (Recommended)**

1. Go to https://railway.app/dashboard
2. Create new project: `velkor-platform`
3. Add services:
   - PostgreSQL (managed database)
   - Strapi (Node.js from GitHub)
   - Next.js (Node.js from GitHub)

**Option B: Using Railway CLI**

```bash
# Add PostgreSQL
railway add --name postgres

# Deploy
railway up
```

### Step 4: Connect GitHub Repository

1. Go to Railway dashboard
2. Select `velkor-platform` project
3. Project Settings → GitHub Repository
4. Connect: `CrazyBoyGDL/velkor-platform`
5. Enable auto-deploy on `main` branch

### Step 5: Configure Environment Variables

In Railway Dashboard → Project Variables:

#### Strapi Service
```
DATABASE_CLIENT=postgres
DATABASE_HOST=<railway-postgres-host>
DATABASE_PORT=5432
DATABASE_NAME=velkor_platform
DATABASE_USERNAME=<auto-generated>
DATABASE_PASSWORD=<auto-generated>
JWT_SECRET=<generate-strong-random-key>
STRAPI_ADMIN_PATH=/admin
API_URL=https://velkor-api.railway.app
NODE_ENV=production
```

#### Next.js Service
```
NEXT_PUBLIC_API_URL=https://velkor-api.railway.app
NEXT_PUBLIC_SITE_NAME=Velkor Platform
NODE_ENV=production
```

### Step 6: Deploy

```bash
# Manual deploy
railway up

# Or push to main for auto-deploy
git push origin main
```

```bash

```

## Troubleshooting

### Build fails
```bash
# Check build logs
railway logs --service strapi
railway logs --service nextjs
```

### Database connection errors
- Verify DATABASE_HOST in Environment Variables
- Check DATABASE_PASSWORD matches
- Ensure PostgreSQL service is running

### Deployment stuck
```bash
# Cancel current deployment
railway cancel

# Check status
railway status

# Retry
railway up
```
